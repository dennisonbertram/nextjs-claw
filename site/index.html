How it works

Describe what you want. Point at what's wrong.

The app runs on your laptop. The chat panel spawns the claude CLI as a subprocess — it uses your existing Claude subscription, so there's no API key to manage.

01 / DESCRIBE
Tell Claude what to build.

"Make a SaaS landing page." "Add a pricing tier." Claude reads your code, edits it, streams tool calls live in the chat.

02 / POINT
Click an element to edit it.

Activate pick mode, click anything in the preview. The tag, text, and Tailwind classes drop into your next message as a reference chip.

03 / WATCH
See the change instantly.

Next.js hot reload serves the edited source. No refresh. No build step. The app changes while you watch.

What's inside

Everything you need. Nothing you don't.

Click-to-edit picker

Hover any element, click to capture tag, text, and classes. Attached to your next message as a reference chip.

▢

4 snap states

Rail, default, wide, fullscreen. Drag to resize — snaps to the nearest preset. Panel and preview never fight for space.

⁂

12 template recipes

SaaS landings, dashboards, portfolios, docs, newsletters. Click one, watch Claude assemble it.

⚒

Stack cookbook

Self-hosted Postgres, auth, email, background jobs, S3 uploads, Stripe. 15 opinionated recipes Claude follows on demand.

◉

Streaming tool chips

Grep, Read, Edit, Write — every call streams as a mono-font chip with state-colored left border. Know exactly what Claude is doing.

☗

Protected paths

The agent's system prompt keeps it out of its own source. The chat UI can't edit itself — the app you build is yours alone.
